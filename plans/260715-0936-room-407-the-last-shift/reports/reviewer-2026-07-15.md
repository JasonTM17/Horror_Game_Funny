## Critical Issues

### C1 — Room 407 wall makes the mandatory route impassable

`scripts/world/gameplay-director.gd:76-82` creates an 8 m-wide corridor and an 8 m-wide solid `Room407Wall` at `z=-78`; `scripts/world/gameplay-director.gd:110` places the animated room door at the same plane. Opening the door rotates only the door body—the wall remains solid across the entire corridor. The clue at `z=-94`, chase threshold at `z<-104`, and exit at `z=-128` cannot be reached through production movement.

Fix: split the wall around a collision-safe doorway aperture, then add a physics traversal test and manual walk-through evidence.

### C2 — The first physical gate is bypassable, breaking progression order

`scripts/world/gameplay-director.gd:76-78` leaves the corridor open across roughly 8 m, while `scripts/world/gameplay-director.gd:103,146-152` places only a 2.2 m-wide door at its center. The player can walk around either side and reach every linear story object. `scripts/world/gameplay-director.gd:236-249` also lets memories be collected without checking `memory_loop_started`, `fuse_installed`, or story stage; the radio checks only memory count at `:213-220`.

Fix: build proper wall/aperture geometry for every gate and enforce the same prerequisites in semantic actions. Test wrong-order movement and action calls independently.

### C3 — The committed experience does not support the required 15–20 minutes

The player moves from `z=11` to the exit at `z=-128`: 139 m, or about 53.5 seconds at the 2.6 m/s walk speed (`scripts/player/player-controller.gd:6`) and 32.4 seconds while sprinting. `scripts/world/gameplay-director.gd:100-112` places the required interactions sequentially along that line; most complete immediately. There are no authored observation windows, traversal variants, or timing evidence that add the remaining 12–17 minutes.

Fix: add substantive continuous-space beats—readable exploration branches, occluded hallway transformations, clue observation, low-intensity recovery windows, and a real Room 407 sequence—then record blind/full playthrough timings. Do not pad with forced idle timers.

## High Priority

### H1 — Serialized checkpoint spawn is never consumed

`scripts/autoload/scene-router.gd:26-29` restores `pending_spawn_id`, but `scripts/world/gameplay-director.gd:90-93` always spawns at `(0, 0.02, 11)`. `scripts/world/gameplay-director.gd:41` also overwrites the restored objective with the opening objective. The local failure path at `:297-313` avoids a reload by teleporting to a hard-coded coordinate and immediately restarting the entity; it does not lock input, fade, stop a loop, or guarantee one authoritative restore.

Fix: map stable spawn IDs to validated markers, apply the restored objective after scene construction, and sequence capture as lock → stop/fade → one restore/reload → one entity spawn → unlock.

### H2 — The progression test bypasses the production path it claims to validate

`tests/progression-test.gd:13-31` calls `handle_story_action()` directly, invokes `on_radio_solved()`, manually sets `room_entered`, calls `on_note_closed()`, and invokes private `_start_chase()`. It never moves the player, opens a door through interaction, crosses a threshold, enters the radio code, exercises pause/settings, or reaches the ending. This is why C1/C2 remain green.

Fix: retain focused state tests, but add a smoke harness that uses public interactions and physics positions, plus explicit door-aperture, router-checkpoint, input-lock, radio wrong-answer, ending, and settings persistence suites.

### H3 — The dynamic hallway is label-only

`scripts/world/dynamic-hallway-controller.gd:7-23` creates three labels and changes their text/color. It has no variant roots, occluded transition, safe marker, collision/process switching, or out-of-bounds recovery promised by the design. It also does not reconstruct `variant` from checkpoint flags.

Fix: implement at least three materially different continuous hallway states behind sight-line blockers and restore the active variant deterministically.

### H4 — Phase 6 is marked complete while required settings/audio/accessibility are missing

`scripts/autoload/audio-manager.gd:8-26` creates/reuses only `Master`; there are no Music/SFX/Ambience buses, named loops, positional cues, or chase ambience lifecycle. `scripts/autoload/settings-manager.gd:5-52` stores only sensitivity, FOV, master volume, flicker, and bob. `scenes/ui/settings-panel.tscn:22-73` exposes only sensitivity, FOV, flicker, and bob—no volume, display mode, shake, grain, reduced-flicker intensity, or reset. No shader directory or synchronized phone/radio subtitles exist.

Fix: either implement the Phase 6 contract before release or truthfully reopen/re-scope it with explicit user approval; do not retain a completed status for missing mandatory requirements.

### H5 — Input locks do not lock interaction and Escape has two owners

`scripts/player/player-interaction.gd:10-28` continues prompting and accepting interactions without consulting the actor's `is_input_locked()`. `scripts/player/player-controller.gd:34-35` always treats Escape as pause, while `scripts/ui/note-reader.gd:50-52` treats the same unhandled event as close-note; neither marks it handled. Opening a note/radio can therefore leave world interaction active, and Escape can close a note and pause in the same event.

Fix: gate prompt/interaction on the centralized lock set, give modal UI ownership of Escape, and mark accepted events handled. Add note+pause and radio+pause race tests.

### H6 — Enemy implementation does not meet its navigation/FSM contract

`scripts/world/chase-entity.gd:28-51` steers directly toward the player's global position. It never waits for a navigation map, uses no bounded waypoint fallback, and can stall on authored collision. `STALK` is never entered and `SEARCH` merely targets the latest position overwritten every active frame.

Fix: either implement the planned navigation-readiness/waypoint contract and traverse every chase corner, or simplify/document a genuinely collision-free authored chase lane and remove unreachable FSM claims.

## Medium Priority

### M1 — Ending authorization is weaker than the documented gate

`scripts/world/gameplay-director.gd:229-233` requires only `chase_started`. It does not validate all memories, radio solution, room entry/final clue, or a chase-complete fact as Phase 5 requires.

Fix: make one explicit ending predicate and test every missing prerequisite independently.

### M2 — `show_objective` is a dead input action

`project.godot:71-74` defines Tab as `show_objective`, but no committed script consumes it. The README advertises “Review objective | Tab.”

Fix: implement objective review behavior or remove the advertised mapping.
